# 07 プロダクトロードマップ会議

## 目的
ロードマップ項目の優先順位と、次リリースで実施する範囲を確定する。

## コピー用AIワークオーダー
```text
会議タイプ: プロダクトロードマップ会議
対象期間: [次四半期]
対象プロダクト: [プロダクト名]

以下の順で出力してください。
1. ロードマップ候補一覧（価値/コスト/リスク）
2. 優先順位の根拠（顧客価値, 売上影響, 開発負荷）
3. 次リリース採用項目
4. 見送り項目と再評価条件
5. 実行タスク（担当/期限/完了条件）
6. 不足情報
7. 上司向け3行報告
```

## 必須確認事項
- 優先順位の判断軸が明示されているか
- 技術負債対応が完全に抜け落ちていないか
- 見送り項目に再評価条件があるか

## 上司向け3行報告フォーマット
- 事実: [次リリース採用項目]
- 課題: [優先順位の競合点]
- 対応: [決定した実行スコープ]
