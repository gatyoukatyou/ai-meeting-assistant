# 08 障害レビュー / ポストモーテム

## 目的
障害の再発防止に必要な原因分析と是正計画を、責任追及ではなく仕組み改善として整理する。

## コピー用AIワークオーダー
```text
会議タイプ: 障害レビュー / ポストモーテム
対象障害: [障害名/発生日]

以下の順で出力してください。
1. 事象サマリー（発生〜復旧の時系列）
2. 影響範囲（顧客/売上/内部運用）
3. 根本原因（人・プロセス・技術で分離）
4. 再発防止策（短期/中期）
5. フォローアップタスク（担当/期限/完了条件）
6. 不足情報
7. 上司向け3行報告
```

## 必須確認事項
- タイムラインに時刻欠落がないか
- 「直接原因」と「根本原因」が分離されているか
- 再発防止策が監視・運用・実装の3層で設計されているか

## 上司向け3行報告フォーマット
- 事実: [障害影響と復旧時刻]
- 課題: [根本原因]
- 対応: [再発防止の最優先タスク]
