# 05 営業パイプライン会議

## 目的
案件パイプラインのボトルネックを特定し、受注確度を上げる行動を確定する。

## コピー用AIワークオーダー
```text
会議タイプ: 営業パイプライン会議
対象期間: [週次]
対象チーム: [チーム名]

以下の順で出力してください。
1. パイプラインサマリー（件数/金額/ステージ）
2. 停滞案件トップ5（停滞日数つき）
3. 失注リスク案件（理由別）
4. 今週の受注最大化アクション
5. アクション一覧（担当/期限/完了条件）
6. 不足情報
7. 上司向け3行報告
```

## 必須確認事項
- 金額ベースと件数ベースの議論が混ざっていないか
- 停滞案件に「次の顧客接点日」が入っているか
- 受注見込み確度の根拠が明示されているか

## 上司向け3行報告フォーマット
- 事実: [パイプライン総額と停滞件数]
- 課題: [失注リスクの主因]
- 対応: [今週の重点アクション]
